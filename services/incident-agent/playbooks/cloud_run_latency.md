# Cloud Run レイテンシ調査

## 最初に見るべき箇所
1. リクエストレイテンシと5xx率の変化を確認する。
2. revision切替時刻、インスタンス数、コールドスタートの相関を見る。
3. WARNING以上のログからCloud SQLや外部APIの待ち時間を確認する。

## 既知の原因
- 新revisionの性能回帰
- Cloud SQL接続プール枯渇
- 外部APIの遅延
- 最小インスタンス不足によるコールドスタート

## 確信の判断基準
- 異常開始がrevision切替または依存先エラーと時刻一致する。
- レイテンシ上昇を裏付けるログまたは依存メトリクスがある。
