# mogu 確定仕様(壁打ち反映版 v1)

> 本書はチーム共通の拠り所。用語・画面・型・ガードレールはここを正とする。

---

## 1. 用語集(全画面・ERD・会話で統一)

| 用語 | 定義 | Spotify対応 |
| --- | --- | --- |
| **スポット** | 1軒のお店。place_id参照+個人の付与情報(写真・一言・3段階評価・タグ) | 曲 |
| **コレクション** | 名前を付けたスポットの一覧。編集=自己表現 | プレイリスト |
| **マイページ** | 自分のコレクションと記録の置き場 | ライブラリ |
| **リコレクション** | 友達のスポットを自分のコレクションに追加する行為。出典(via)が自動で紐づく | 保存/リピン |
| **フラグ(静かな承認)** | リコレクション発生時に原作者へ届く通知。本人にのみ見える | — |

## 2. 決定事項(この壁打ちで確定)

- リコレクションの単位は**スポット(カード)単位**。コレクションごとの複製はしない
- ホームは**永続フィード型**。ストーリー形式(24時間消滅)は不採用 — 良い推薦は常緑コンテンツであり、FOMO装置はアテンション・デトックス方針と矛盾するため
- タグは**2層設計**: 構造化タグ(エリア/ジャンル/シチュエーション = S1検索チップと同語彙 = エージェントの検索キー) + 自由タグ(「半個室」「静か」等、聞き上手エージェント生成含む)
- **数字の主語ルール**: 人に付く数字(獲得フラグ数)は本人のみ表示 / 店に付く数字(「輪でn人が保存」)は表示OK(意思決定シグナル)
- **店舗側との連携はしない**(広告・掲載費・送客手数料を受け取らない)。推薦の中立性を構造で担保 = ノイズ防御第2層
- **圧力型ゲーミフィケーション(ストリーク・連続記録バッジ等)は不採用**
- エージェントが**コレクション末尾に「このコレクションに合いそうなお店」を提案**(Spotifyのプレイリスト末尾おすすめの写像。育てる労力を削る+基準①)
- 将来枠: **食のWrapped**(月次振り返りカード、LINE共有向き) / **共同コレクション**(幹事エージェントへの布石)

## 3. 改訂ユーザーストーリー(確定用語版)

1. Aさんはデートの店を探すとき、まず自分のコレクションから探す
2. 見つからなければ友達Bさんのコレクションを見る
3. 良いスポットがあれば1タップでリコレクション(via B が自動付与)
4. Bさんに静かなフラグが届く(本人にのみ見える)
5. CさんがAさんのコレクション内の元Bさん発のスポットをリコレクションすると、原作者Bさんにフラグが届く(recollection_edges.origin_user)。2次以降は匿名クレジット、Bさんの写真・コメントは1次の友達まで
6. スポットはデフォルトでGoogle情報(表示時取得)+ 個人の写真・一言・タグを付与できる
7. ソロ期は自分のコレクションを育てる。売り文句は「記録できる」ではなく**「記録が育つ」**(増やすほど断言が鋭くなる = 記録の利子)。記録がアイデンティティになる
8. 検索(今の気持ち)を入力するとエージェントが傾向+蓄積データから断言。ホームにも1日1枚だけ一推しが出る
9. 友達検索で見えるのは名前とアバターのみ(公開されるのは存在、守られるのは中身)。相互承認で初めてコレクションが見える

## 4. コレクションの属性

| 属性 | 必須 | 備考 |
| --- | --- | --- |
| 名前 | ✔ | 絵文字可 |
| カバー画像 | 任意 | **ユーザー自身の写真のみ**。未設定時はテキスト/カラーベースの自動カバー。⚠ Place Photosの合成・加工によるカバー生成は規約違反(派生物作成・コンテンツ加工の禁止)のため実装しない |
| 説明文 | 任意 | |
| 公開範囲 | ✔ | friends(既定) / secret(非伝播・非通知) |
| テーマ | 任意(nullable) | シチュエーション属性(デート/会社/友達等)。付いていればエージェントの横断検索が賢くなる。入力は強制しない |

## 5. ホーム3段構成

| 段 | 役割 | 挙動 | ソロ期の空状態 |
| --- | --- | --- | --- |
| 上: アバター行 | 人への導線 | 新着リング=**既読管理**(24時間で消えない)。新着順。行末に常設「+友達を招待」チップ | 招待チップのみ |
| 中: 一推しカード | 意思決定の即答 | **1枚だけ**。1日1回更新+文脈トリガー(金曜夕方)。タップでS2へ | 「今夜どこ行く？」入力誘導カード(S1導線) |
| 下: 新着フィード | 輪の出来事 | **時系列順・アルゴリズムなし**(設計宣言)。カード=写真カルーセル+推薦者+3段階+一言+リコレクションボタン+「輪でn人が保存」 | 自分の記録+「最初のお店を追加」プロンプト |

**既読管理(MVP)**: サーバ ERD は変更しない。クライアント `localStorage` の `lastReadFeedAt`(ISO8601) で「最後にフィードを見た時刻」を保持し、それ以降の友達 activity で新着リングを点灯する。

下部タブ: ホーム / 友達 / 検索(今の気持ち・中央強調は磨き込みで) / マイページ。
実装注意: `100dvh` + `env(safe-area-inset-bottom)` + `viewport-fit=cover`。PCはmax-width中央寄せ(専用レイアウトは作らない)。

## 6. 型定義(フロント⇔バックの契約。モック実装はこの型で開始)

```typescript
// types.ts — この型がAPI契約。ERDのDDLはこれを満たすように実装する
export type User = {
  id: string              // firebase_uid
  displayName: string
  avatarColor: string
  avatarUrl: string | null  // GCS photo avatar; null → color + initial (#259)
}

export type Spot = {
  id: string
  placeId: string         // Google参照キー(店名・住所は保存しない。表示時取得)
  addedBy: string         // User.id
  collectionId: string
  photoUrls: string[]     // 自前Cloud Storage(ユーザー撮影)。空ならPlace Photosを表示時借用+帰属表示
  comment: string
  rating: 'again' | 'either' | 'no'
  structuredTags: { area?: string; genre?: string; situation?: string }  // 第1層(検索キー)
  freeTags: string[]      // 第2層(聞き上手エージェント生成含む)
  savedCount: number      // 「輪でn人が保存」(店に付く数字=表示OK)
  originUserId: string | null  // リコレクション元の原作者(1次のみ実名、2次以降は匿名表示)
  createdAt: string
}

export type Collection = {
  id: string
  ownerId: string
  name: string
  description: string | null
  coverUrl: string | null      // ユーザー写真のみ。nullならテキスト/カラー自動カバー
  visibility: 'friends' | 'secret'
  theme: string | null         // シチュエーション(任意)
  spotCount: number
}

export type FeedItem = {
  spot: Spot
  actor: Pick<User, 'id' | 'displayName' | 'avatarColor' | 'avatarUrl'>
  collectionName: string
  createdAt: string
}

export type Recommendation = {   // 一推しカード/S2断言
  spot: Spot
  assertion: string              // 断言理由の一文
  evidence: string               // 「Kenが『また行きたい』・輪で4人が保存」
  alternatives: Spot[]           // 折りたたみの残り2件
}

export type FlagNotification = { // 静かな承認(本人のみ)
  type: 'recollected'
  count: number
  isAnonymous: boolean           // depth>=2
  weekOf: string
}
```

## 7. ガードレール(壊すとコンセプトが死ぬ)

1. 数字の主語ルール(人→本人のみ / 店→表示可)
2. コンテンツ伝播は1次限定(2次以降はスポット+匿名クレジットのみ)
3. フィードは時系列・非アルゴリズム
4. FOMO装置(消えるコンテンツ・ストリーク)を作らない
5. 店から金を受け取らない
6. カバー画像にGoogle素材を加工・合成しない(ユーザー写真 or 自前生成のみ)
7. Google情報は保存せず表示時取得+帰属表示(既定の参照モデル)
